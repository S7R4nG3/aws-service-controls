Encode the selected FSx type as the standard for specific workload categories in IaC module variables and enforce through pipeline guardrails. |  |
| HIGH | COST-FSx-003 | AWS FSx Cost Optimization Best Practices 2024.1 | Use FSx for Lustre scratch file systems for transient workloads | FSx for Lustre scratch file systems are significantly cheaper than persistent file systems and are appropriate for short-lived, compute-intensive workloads such as HPC jobs or batch processing where data durability is not required. | **IaC** - In IaC templates for batch or HPC workloads, specify SCRATCH_2 deployment type for FSx for Lustre. | HIGH | COST-FSx-004 | AWS FSx Cost Optimization Best Practices 2024.1 | Delete unused FSx file systems and backups | Orphaned FSx file systems and accumulated backups that are no longer needed continue to incur storage charges. | MEDIUM | COST-FSx-005 | AWS FSx Cost Optimization Best Practices 2024.1 | Optimize FSx backup retention periods | Excessive backup retention periods generate unnecessary storage costs. | MEDIUM | COST-FSx-007 | AWS FSx Cost Optimization Best Practices 2024.1 | Choose the appropriate FSx for Lustre storage type (SSD vs HDD) | FSx for Lustre persistent file systems can use SSD or HDD storage.
